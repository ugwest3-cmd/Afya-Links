Revenue Structure
• Pharmacy commission → % of drug order value
• Driver commission → 15% of delivery fee
Platform earns from both sides.
🔷 FULL ORDER FLOW (DETAILED)
1️⃣ ORDER CREATION
Clinic creates order.
Example:
OrderValue (Drugs) = 200,000 UGX DeliveryFee = 10,000 UGX PharmacyCommissionRate = 8% DriverCommissionRate = 15% 
System calculates:
Pharmacy side
PharmacyCommission = 16,000 PharmacyNet = 184,000 
Driver side
DriverCommission = 1,500 DriverNet = 8,500 
Total platform earnings
TotalPlatformRevenue = 16,000 + 1,500 = 17,500 
Total clinic payment
TotalPayable = 200,000 + 10,000 = 210,000 
Order State:
AWAITING_PAYMENT EscrowStatus = NOT_FUNDED 
2️⃣ PAYMENT VIA PESAPAL
Clinic pays 210,000 through Pesapal checkout.
Pesapal webhook returns:
PaymentStatus = COMPLETED TransactionID = PESA-XXXX AmountReceived = 210,000 
System validates:
• Amount matches TotalPayable
• OrderID matches
System updates:
PaymentStatus = VERIFIED EscrowStatus = LOCKED OrderStatus = PAID_READY FundsLocked = 210,000 
Money is now logically escrowed inside your system.
3️⃣ PHARMACY PREPARATION
Pharmacy receives notification:
Order AFY-0001 PAID. Prepare items. 
After preparation:
OrderStatus = READY_FOR_PICKUP 
4️⃣ DRIVER ACCEPTS DELIVERY
Driver accepts job.
System logs:
DriverID PickupTime ExpectedDeliveryTime 
Order becomes:
OrderStatus = OUT_FOR_DELIVERY 
5️⃣ DELIVERY CONFIRMATION (TRIGGER EVENT)
Driver arrives at clinic.
To prevent fraud:
Step A — OTP Verification
System sends OTP to clinic phone.
Clinic enters OTP in app.
If correct:
DeliveryConfirmation = VALID Timestamp recorded 
6️⃣ AUTOMATIC ESCROW RELEASE (ATOMIC TRANSACTION)
System executes:
BEGIN TRANSACTION Release PharmacyNet = 184,000 Release DriverNet = 8,500 Retain PharmacyCommission = 16,000 Retain DriverCommission = 1,500 EscrowStatus = RELEASED OrderStatus = COMPLETED PayoutStatus = INITIATED COMMIT 
7️⃣ PAYOUT EXECUTION
Option A (MVP Phase)
You send:
• 184,000 to pharmacy
• 8,500 to driver
Immediately after confirmation.
Option B (Automated Later)
System triggers payout API.

🔷 EDGE CASE HANDLING
❌ Payment Fails
OrderStatus = PAYMENT_FAILED EscrowStatus = NOT_FUNDED 
No processing allowed.
❌ Driver Fails to Deliver
OrderStatus = DELIVERY_FAILED EscrowStatus = LOCKED 
Admin review required.
❌ Clinic Refuses Delivery
OrderStatus = DISPUTE EscrowStatus = LOCKED 
Funds remain locked.
Manual resolution required.
❌ Pharmacy Cancels After Payment
Initiate Pesapal refund.
OrderStatus = REFUNDED EscrowStatus = RETURNED 
🔷 IMPORTANT CONTROL RULES
• Delivery confirmation must come from clinic device.
• Driver cannot confirm delivery.
• OTP must expire after X minutes.
• All state transitions must be sequential (no skipping).
• Funds cannot release if EscrowStatus ≠ LOCKED.
